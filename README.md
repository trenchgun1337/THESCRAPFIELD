# Repository Structure

This repository uses two separate branches to organize its content and keep the project clean and maintainable.

## Branch Overview

### `main`
The **main branch** is used exclusively for storing project resources and data files.

This includes, but is not limited to:

- Images
- GIFs
- Videos
- Textures
- Visual assets
- Media resources
- Reference files
- Archived data
- Other non-code materials

This branch acts as a **central storage for all project assets**.

### `code`
The **code branch** is dedicated strictly to the website or project source code.

Only programming-related files should be stored here, such as:

- HTML
- CSS
- JavaScript
- Configuration files
- Framework files
- Scripts
- Build files

No media assets or large resource files should be stored in this branch unless they are **directly required by the code**.

## Purpose of This Structure

Separating assets and source code helps to:

- Keep the codebase lightweight and organized
- Prevent unnecessary asset files from cluttering the code branch
- Make deployments easier and faster
- Improve repository navigation and maintenance

## Important Notes

- **Do not upload code files to the `main` branch.**
- **Do not upload unrelated assets to the `code` branch.**

Each branch has a **clear and specific purpose**, and maintaining this separation ensures better project organization.
